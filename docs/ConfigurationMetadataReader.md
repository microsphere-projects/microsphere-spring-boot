# ConfigurationMetadataReader

## Overview

`ConfigurationMetadataReader` reads Spring Boot configuration metadata from `spring-configuration-metadata.json` and `additional-spring-configuration-metadata.json` files on the classpath. These files are typically generated by the `spring-boot-configuration-processor` annotation processor.

The reader uses `ResourcePatternResolver` to discover all metadata files across the classpath (including from dependencies) and parses them into a unified `ConfigurationMetadata` object.

## Package

`io.microsphere.spring.boot.context.properties.metadata`

## Since

`1.0.0`

## API Details

### Class: `ConfigurationMetadataReader`

**Implements:** `ResourceLoaderAware`

### Constants

| Constant | Type | Value | Description |
|---|---|---|---|
| `METADATA_PATH` | `String` | `"classpath*:/META-INF/spring-configuration-metadata.json"` | Path pattern for standard metadata |
| `ADDITIONAL_METADATA_PATH` | `String` | `"classpath*:/META-INF/additional-spring-configuration-metadata.json"` | Path pattern for additional metadata |
| `BEAN_NAME` | `String` | `"originTrackedConfigurationPropertyInitializer"` | Default bean name |

### Methods

```java
public ConfigurationMetadata read()
```
Reads and merges all configuration metadata files from the classpath.

```java
public void setResourceLoader(ResourceLoader resourceLoader)
```
Sets the `ResourceLoader` used to discover metadata files.

```java
public ResourcePatternResolver getResourcePatternResolver()
```
Returns the `ResourcePatternResolver` used internally.

## Example Code

```java
ConfigurationMetadataReader reader = new ConfigurationMetadataReader();
reader.setResourceLoader(new DefaultResourceLoader());

ConfigurationMetadata metadata = reader.read();
metadata.getItems().forEach(item ->
    System.out.println(item.getName() + " = " + item.getType())
);
```
